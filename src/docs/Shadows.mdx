# Shadows

Shadow tokens provide elevation cues. They are defined in `src/theme/tokens.scss` and exposed as CSS custom properties (`--gs-shadow-*`). Dark mode uses stronger shadows to compensate for darker backgrounds.

### When to Use Each Level

- **sm**: Default card elevation. Most cards in the UI sit at this level. The shadow is barely visible and just lifts the card from the page background.
- **md**: Interactive states. When a card is hovered or a dropdown opens, use md shadow to indicate interactivity.
- **lg**: Overlays that need clear separation from the page. Modals, bottom sheets, and popovers.
- **xl**: Maximum elevation. Used for the Storybook mobile frame wrapper and full-screen overlays.
- **bottom-bar**: Specialized shadow that casts upward. Used exclusively for fixed bottom navigation (TabBar, ActionBar).

<SectionTitle>SCSS Usage</SectionTitle>

```scss
@use '../../../theme/tokens' as *;

.card {
  box-shadow: $shadow-sm;

  &:hover {
    box-shadow: $shadow-md;
  }
}

.modal {
  box-shadow: $shadow-lg;
}

.bottom-bar {
  box-shadow: $shadow-bottom-bar;
}
```

For theme-aware shadows (that adjust automatically in dark mode), use the CSS custom properties:

```scss
.card {
  box-shadow: var(--gs-shadow-sm);

  &:hover {
    box-shadow: var(--gs-shadow-md);
  }
}
```
