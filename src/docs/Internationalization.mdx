# Internationalization (i18n)

`@ansoragroup/ui` ships with built-in localization for Uzbek, Russian, and English. All user-facing strings in components are translatable, and prices format automatically based on locale and currency settings.

---

<SectionTitle>Quick Start</SectionTitle>

Wrap your application with `GeekShopProvider` to configure locale and currency:

```tsx
import { GeekShopProvider } from '@ansoragroup/ui'

function App() {
  return (
    <GeekShopProvider locale="uz" currency="UZS">
      <YourApp />
    </GeekShopProvider>
  )
}
```

All components inside the provider will render in the specified locale and format prices in the specified currency.

<SectionTitle>Translation Function: t()</SectionTitle>

Inside components, use the `useGeekShop` hook to access the `t()` function:

```tsx
import { useGeekShop } from '@ansoragroup/ui'

function MyComponent() {
  const { t } = useGeekShop()

  return (
    <div>
      <h1>{t('page.home')}</h1>
      <button>{t('commerce.addToCart')}</button>
    </div>
  )
}
```

### Interpolation

Translation strings support `{variable}` placeholders:

```tsx
// Template: 'commerce.inStock' = '{count} in stock'
t('commerce.inStock', { count: 5 })
// => "5 in stock" (en)
// => "5 dona bor" (uz)
// => "5 в наличии" (ru)

// Template: 'commerce.selected' = 'Selected: {count}'
t('commerce.selected', { count: 3 })
// => "Selected: 3" (en)

// Template: 'product.installment' = '{price}/mo'
t('product.installment', { price: '50 000' })
// => "50 000/mo" (en)
```

If a key is not found, the key itself is returned unchanged. This makes it easy to spot missing translations during development.

<SectionTitle>Price Formatting: formatPrice()</SectionTitle>

The `formatPrice` function from the hook formats numbers according to the configured locale and currency:

```tsx
import { useGeekShop } from '@ansoragroup/ui'

function PriceTag() {
  const { formatPrice } = useGeekShop()

  return (
    <div>
      {/* Uses provider's locale and currency */}
      <span>{formatPrice(5200000)}</span>
      {/* => "5 200 000 so'm" (uz/UZS) */}
      {/* => "5 200 000 сум" (ru/UZS) */}
      {/* => "5,200,000 UZS" (en/UZS) */}

      {/* Override currency for a specific call */}
      <span>{formatPrice(49.99, { currency: 'USD' })}</span>
      {/* => "$49.99" */}

      {/* Hide currency symbol */}
      <span>{formatPrice(5200000, { showCurrency: false })}</span>
      {/* => "5 200 000" */}
    </div>
  )
}
```

### Standalone formatPrice

You can also import `formatPrice` as a standalone utility (without provider):

```tsx
import { formatPrice } from '@ansoragroup/ui'

formatPrice(5200000, 'UZS', 'uz')
// => "5 200 000 so'm"

formatPrice(49.99, 'USD', 'en')
// => "$49.99"

formatPrice(4500, 'RUB', 'ru')
// => "4 500 ₽"
```

<SectionTitle>Using Without Provider</SectionTitle>

All components work without wrapping in `GeekShopProvider`. They will use the defaults:
- **Locale:** `uz` (Uzbek)
- **Currency:** `UZS`

```tsx
// No provider needed for basic usage
import { PriceDisplay, ActionBar } from '@ansoragroup/ui'

function Page() {
  return (
    <div>
      {/* Renders in Uzbek by default */}
      <PriceDisplay price={299000} />
      <ActionBar onCartClick={() => {}} onBuyClick={() => {}} />
    </div>
  )
}
```

<SectionTitle>Custom Translations</SectionTitle>

Override specific translation keys by passing a `translations` prop to the provider:

```tsx
import { GeekShopProvider } from '@ansoragroup/ui'

const customTranslations = {
  en: {
    'commerce.addToCart': 'Add to Bag',    // override
    'commerce.buyNow': 'Purchase Now',     // override
    'custom.myKey': 'My custom string',    // add new key
  },
  uz: {
    'commerce.addToCart': "Sumkaga qo'shish",
  },
}

function App() {
  return (
    <GeekShopProvider locale="en" currency="USD" translations={customTranslations}>
      <YourApp />
    </GeekShopProvider>
  )
}
```

Custom translations are merged with built-in ones. Your overrides take precedence.

<SectionTitle>Next.js Integration</SectionTitle>

### App Router

```tsx
// app/providers.tsx
'use client'

import { GeekShopProvider } from '@ansoragroup/ui'

export function Providers({ locale, children }: { locale: 'uz' | 'ru' | 'en'; children: React.ReactNode }) {
  return (
    <GeekShopProvider locale={locale} currency="UZS">
      {children}
    </GeekShopProvider>
  )
}
```

```tsx
// app/[locale]/layout.tsx
import '@ansoragroup/ui/styles'
import { Providers } from '../providers'

export default function Layout({ children, params }: { children: React.ReactNode; params: { locale: string } }) {
  return (
    <html lang={params.locale}>
      <body>
        <Providers locale={params.locale as 'uz' | 'ru' | 'en'}>
          {children}
        </Providers>
      </body>
    </html>
  )
}
```

### Dynamic Locale Switching

```tsx
'use client'

import { useState } from 'react'
import { GeekShopProvider, LanguageSwitcher, CurrencySwitcher } from '@ansoragroup/ui'
import type { Locale, CurrencyCode } from '@ansoragroup/ui'

function App() {
  const [locale, setLocale] = useState<Locale>('uz')
  const [currency, setCurrency] = useState<CurrencyCode>('UZS')

  return (
    <GeekShopProvider locale={locale} currency={currency}>
      <LanguageSwitcher value={locale} onChange={setLocale} />
      <CurrencySwitcher value={currency} onChange={setCurrency} />
      <YourApp />
    </GeekShopProvider>
  )
}
```
