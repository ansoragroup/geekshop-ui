<SectionTitle>Form + FormItem (Built-in Validation)</SectionTitle>

The `Form` component provides a validation context, and `FormItem` wraps individual fields with labels, error messages, and help text. This is the simplest way to build validated forms without any external library.

<SubTitle>Basic Usage</SubTitle>

```tsx
import { useRef, useState } from 'react'
import { Form, FormItem, Input, Button } from '@ansoragroup/ui'
import type { FormHandle } from '@ansoragroup/ui'

function RegisterForm() {
  const formRef = useRef<FormHandle>(null)
  const [name, setName] = useState('')
  const [email, setEmail] = useState('')

  return (
    <Form
      ref={formRef}
      onSubmit={(values) => console.log('Submitted:', values)}
      onError={(errors) => console.log('Errors:', errors)}
      initialValues={{ name: '', email: '' }}
    >
      <FormItem name="name" label="Name" required>
        <Input
          placeholder="John Doe"
          value={name}
          onChange={(v) => {
            setName(v)
            formRef.current?.setFieldValue('name', v)
          }}
        />
      </FormItem>

      <FormItem name="email" label="Email" required>
        <Input
          placeholder="you@example.com"
          value={email}
          onChange={(v) => {
            setEmail(v)
            formRef.current?.setFieldValue('email', v)
          }}
        />
      </FormItem>

      <Button variant="primary" size="lg" block type="submit">
        Register
      </Button>
    </Form>
  )
}
```

<SubTitle>Imperative Form Handle</SubTitle>

Access the Form's imperative methods via `ref`:

```tsx
import { useRef } from 'react'
import type { FormHandle } from '@ansoragroup/ui'

const formRef = useRef<FormHandle>(null)

// Submit programmatically
formRef.current?.submit()

// Reset to initial values
formRef.current?.reset()

// Validate without submitting (returns errors object)
const errors = formRef.current?.validate()

// Set a single field value
formRef.current?.setFieldValue('email', 'new@example.com')

// Get all current values
const values = formRef.current?.getValues()
```

<SectionTitle>Controlled vs. Uncontrolled</SectionTitle>

All form components support both **controlled** (you manage state) and **uncontrolled** (component manages state) usage via the `useControllableState` hook.

<SubTitle>Uncontrolled (simplest)</SubTitle>

Just provide `defaultValue`. The component manages its own state internally:

```tsx
import { Input, Checkbox, Switch } from '@ansoragroup/ui'

function SimpleForm() {
  return (
    <form>
      {/* Component manages its own value */}
      <Input label="Email" defaultValue="" />
      <Checkbox defaultChecked={false}>Subscribe to newsletter</Checkbox>
      <Switch defaultChecked={true}>Dark mode</Switch>
    </form>
  )
}
```

<SubTitle>Controlled (full control)</SubTitle>

Provide `value` + `onChange`. You manage the state:

```tsx
import { useState } from 'react'
import { Input, Checkbox, Switch } from '@ansoragroup/ui'

function ControlledForm() {
  const [email, setEmail] = useState('')
  const [subscribe, setSubscribe] = useState(false)
  const [darkMode, setDarkMode] = useState(true)

  return (
    <form>
      <Input label="Email" value={email} onChange={setEmail} />
      <Checkbox checked={subscribe} onChange={setSubscribe}>Subscribe</Checkbox>
      <Switch checked={darkMode} onChange={setDarkMode}>Dark mode</Switch>
    </form>
  )
}
```

<SubTitle>How useControllableState Works</SubTitle>

The hook powers all form components internally. It determines whether the component is controlled or uncontrolled based on whether `value` is provided:

```tsx
import { useControllableState } from '@ansoragroup/ui'

function MyComponent({ value, defaultValue, onChange }) {
  const [internalValue, setInternalValue] = useControllableState({
    value,                              // if provided, component is controlled
    defaultValue: defaultValue ?? '',   // fallback for uncontrolled
    onChange,                           // called on every change
  })

  // Use internalValue and setInternalValue everywhere
  // The hook handles both modes transparently
}
```

<SectionTitle>Form Field Anatomy</SectionTitle>

A complete form field consists of four parts:

```
+-----------------------------------------+
|  Label *                                |   <- Associates with input via htmlFor/id
+-----------------------------------------+
|  [ Input field                        ] |   <- The interactive control
+-----------------------------------------+
|  Error message or helper text           |   <- Contextual guidance
+-----------------------------------------+
```

The `FormItem` component provides label, required asterisk, error display, and help text for any child component:

```tsx
<FormItem
  name="email"
  label="Email Address"
  required
  helpText="We'll never share your email"
  rules={[{ type: 'pattern', value: /email-regex/, message: 'Invalid email' }]}
>
  <Input placeholder="you@example.com" value={email} onChange={setEmail} />
</FormItem>
```

When validation fails, the error message replaces the helper text and the `role="alert"` attribute ensures screen readers announce the error.

<SubTitle>Inline Validation (on blur)</SubTitle>

Validate each field as the user leaves it, without the `Form` wrapper:

```tsx
import { useState } from 'react'
import { Input, Button } from '@ansoragroup/ui'

function LoginForm() {
  const [email, setEmail] = useState('')
  const [emailError, setEmailError] = useState('')

  const validateEmail = () => {
    if (!email) {
      setEmailError('Email is required')
    } else if (!email.includes('@')) {
      setEmailError('Please enter a valid email')
    } else {
      setEmailError('')
    }
  }

  return (
    <form>
      <Input
        label="Email"
        value={email}
        onChange={setEmail}
        onBlur={validateEmail}
        error={emailError}
      />
      <Button variant="primary" size="lg" block type="submit">
        Sign In
      </Button>
    </form>
  )
}
```

<SubTitle>On-Submit Validation</SubTitle>

Validate all fields when the form is submitted:

```tsx
import { useState } from 'react'
import { Input, Button } from '@ansoragroup/ui'

function RegisterForm() {
  const [fields, setFields] = useState({ email: '', password: '' })
  const [errors, setErrors] = useState<Record<string, string>>({})

  const handleSubmit = (e: React.FormEvent) => {
    e.preventDefault()
    const newErrors: Record<string, string> = {}

    if (!fields.email) newErrors.email = 'Email is required'
    if (!fields.password) newErrors.password = 'Password is required'
    if (fields.password.length < 8) newErrors.password = 'Minimum 8 characters'

    setErrors(newErrors)
    if (Object.keys(newErrors).length === 0) {
      // Submit form
    }
  }

  return (
    <form onSubmit={handleSubmit}>
      <Input
        label="Email"
        value={fields.email}
        onChange={(v) => setFields(f => ({ ...f, email: v }))}
        error={errors.email}
      />
      <Input
        label="Password"
        type="password"
        value={fields.password}
        onChange={(v) => setFields(f => ({ ...f, password: v }))}
        error={errors.password}
      />
      <Button variant="primary" size="lg" block type="submit">
        Register
      </Button>
    </form>
  )
}
```

<SectionTitle>Third-Party Form Libraries</SectionTitle>

All form components spread `...rest` props onto their root or input element, making them fully compatible with external form libraries. You do NOT need to use our `Form` / `FormItem` wrappers -- the individual components work standalone.

<SubTitle>react-hook-form</SubTitle>

<InfoBox type="tip">
  <strong>Recommended for complex forms.</strong> react-hook-form provides superior performance by minimizing re-renders. Use the <code>Controller</code> component for components with custom <code>onChange</code> signatures.
</InfoBox>

```tsx
import { useForm, Controller } from 'react-hook-form'
import { Input, Select, Checkbox, Button } from '@ansoragroup/ui'

interface CheckoutFormValues {
  name: string
  email: string
  city: string
  agreeToTerms: boolean
}

function CheckoutForm() {
  const { control, handleSubmit, formState: { errors } } = useForm<CheckoutFormValues>({
    defaultValues: {
      name: '',
      email: '',
      city: '',
      agreeToTerms: false,
    },
  })

  const onSubmit = (data: CheckoutFormValues) => {
    console.log('Order data:', data)
  }

  return (
    <form onSubmit={handleSubmit(onSubmit)} style={{ display: 'flex', flexDirection: 'column', gap: 16 }}>
      <Controller
        name="name"
        control={control}
        rules={{ required: 'Name is required' }}
        render={({ field }) => (
          <Input
            label="Ism"
            placeholder="John Doe"
            value={field.value}
            onChange={field.onChange}
            onBlur={field.onBlur}
            error={errors.name?.message}
          />
        )}
      />

      <Controller
        name="email"
        control={control}
        rules={{
          required: 'Email is required',
          pattern: {
            value: /^[^\s@]+@[^\s@]+\.[^\s@]+$/,
            message: 'Invalid email',
          },
        }}
        render={({ field }) => (
          <Input
            label="Email"
            type="email"
            placeholder="you@example.com"
            value={field.value}
            onChange={field.onChange}
            onBlur={field.onBlur}
            error={errors.email?.message}
          />
        )}
      />

      <Controller
        name="city"
        control={control}
        rules={{ required: 'Please select a city' }}
        render={({ field }) => (
          <Select
            label="Shahar"
            options={[
              { value: 'tashkent', label: 'Toshkent' },
              { value: 'samarkand', label: 'Samarqand' },
              { value: 'bukhara', label: 'Buxoro' },
            ]}
            value={field.value}
            onChange={field.onChange}
            error={errors.city?.message}
          />
        )}
      />

      <Controller
        name="agreeToTerms"
        control={control}
        rules={{ required: 'You must agree to the terms' }}
        render={({ field }) => (
          <Checkbox
            checked={field.value}
            onChange={field.onChange}
            label="I agree to the Terms of Use"
          />
        )}
      />

      <Button variant="primary" size="lg" block type="submit">
        Place Order
      </Button>
    </form>
  )
}
```

<SubTitle>Formik</SubTitle>

```tsx
import { Formik, Form as FormikForm, Field } from 'formik'
import { Input, Select, Button } from '@ansoragroup/ui'

interface FormValues {
  name: string
  email: string
  city: string
}

function FormikCheckout() {
  return (
    <Formik<FormValues>
      initialValues={{ name: '', email: '', city: '' }}
      validate={(values) => {
        const errors: Partial<Record<keyof FormValues, string>> = {}
        if (!values.name) errors.name = 'Name is required'
        if (!values.email) errors.email = 'Email is required'
        if (values.email && !values.email.includes('@')) errors.email = 'Invalid email'
        return errors
      }}
      onSubmit={(values) => console.log('Formik submit:', values)}
    >
      {({ values, errors, touched, handleChange, handleBlur, setFieldValue }) => (
        <FormikForm style={{ display: 'flex', flexDirection: 'column', gap: 16 }}>
          <Input
            label="Ism"
            name="name"
            placeholder="John Doe"
            value={values.name}
            onChange={(v) => setFieldValue('name', v)}
            onBlur={handleBlur}
            error={touched.name ? errors.name : undefined}
          />

          <Input
            label="Email"
            name="email"
            type="email"
            placeholder="you@example.com"
            value={values.email}
            onChange={(v) => setFieldValue('email', v)}
            onBlur={handleBlur}
            error={touched.email ? errors.email : undefined}
          />

          <Select
            label="Shahar"
            options={[
              { value: 'tashkent', label: 'Toshkent' },
              { value: 'samarkand', label: 'Samarqand' },
            ]}
            value={values.city}
            onChange={(v) => setFieldValue('city', v)}
            error={touched.city ? errors.city : undefined}
          />

          <Button variant="primary" size="lg" block type="submit">
            Submit
          </Button>
        </FormikForm>
      )}
    </Formik>
  )
}
```

<InfoBox type="info">
  Both integrations work because our components accept <code>name</code>, <code>value</code>, <code>onChange</code>, <code>onBlur</code> props, spread additional props (<code>data-*</code>, <code>aria-*</code>, <code>ref</code>) onto the underlying element, and use <code>forwardRef</code> for ref access.
</InfoBox>

<SectionTitle>Error State Handling</SectionTitle>

Form components indicate errors through:

1. **Red border** -- Input/TextArea border color changes to `$color-error`
2. **Error message** -- Text below the field in red
3. **ARIA attributes** -- `aria-invalid="true"` and `aria-describedby` linking to the error message

```tsx
// Error state
<Input
  label="Email"
  value=""
  error="Email is required"        // Triggers red border + error text
/>

// Valid state (no error prop or empty string)
<Input
  label="Email"
  value="user@example.com"
  error=""                          // No error shown
/>
```

When using `FormItem`, error display is automatic:

```tsx
// FormItem automatically shows error text with role="alert"
<FormItem name="email" label="Email" required>
  <Input value={email} onChange={setEmail} />
</FormItem>
// On validation failure -> shows red error text below the input
```

<SectionTitle>Disabled State Handling</SectionTitle>

All form components support a `disabled` prop:

```tsx
<Input label="Email" disabled value="locked@example.com" />
<Checkbox disabled checked>Accepted terms</Checkbox>
<Switch disabled checked>Feature flag</Switch>
<Radio disabled value="option1">Option 1</Radio>
<Button disabled>Submit</Button>
```

Disabled components:
- Reduce opacity to 0.5
- Show `cursor: not-allowed`
- Do not fire `onChange` or `onClick`
- Set `aria-disabled="true"` and the native `disabled` attribute
