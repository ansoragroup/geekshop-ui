# Bundle Size

`@ansoragroup/ui` is built with **Vite in library mode** using `preserveModules`, which means each component is output as a separate module. This enables your bundler to **tree-shake** unused components automatically.

---

<SectionTitle>How Tree-Shaking Works</SectionTitle>

The library outputs ES modules with `preserveModules`, meaning each component gets its own file in the `dist/` directory. When you import specific components, your bundler (Vite, Webpack, Rollup, etc.) only includes what you actually use.

```tsx
// Only Button and its dependencies are bundled (~2 KB)
import { Button } from '@ansoragroup/ui'

// Button + Input are bundled (~4 KB)
import { Button, Input } from '@ansoragroup/ui'

// Both forms are equivalent — tree-shaking works either way
import { Button } from '@ansoragroup/ui'
// is the same as
import { Button } from '@ansoragroup/ui/dist/es/components/form/Button'
```

<InfoBox type="tip">
  You do <strong>not</strong> need deep imports like <code>@ansoragroup/ui/dist/es/components/form/Button</code>. The barrel export at <code>@ansoragroup/ui</code> is fully tree-shakeable. Use the simple import.
</InfoBox>

<SectionTitle>Styles</SectionTitle>

The library uses **CSS Modules** compiled to standard CSS. When you import `@ansoragroup/ui/styles`, you get the complete stylesheet including the CSS custom properties (theme tokens) and all component styles.

```tsx
// In your app's entry point
import '@ansoragroup/ui/styles'
```

<InfoBox type="info">
  The styles import includes <strong>all</strong> component CSS regardless of which components you use. This is expected for a design system -- CSS is highly compressible and the total gzip size is small. Future versions may support CSS-in-JS extraction for per-component styles.
</InfoBox>

<SectionTitle>Optimization Tips</SectionTitle>

<SubTitle>1. Use named imports (default behavior)</SubTitle>

```tsx
// Tree-shakeable -- only imports Button
import { Button } from '@ansoragroup/ui'
```

<SubTitle>2. Lazy-load heavy components</SubTitle>

For components used in modals or below-the-fold, use React.lazy:

```tsx
import { lazy, Suspense } from 'react'
import { Loading } from '@ansoragroup/ui'

const ProductImageGallery = lazy(() =>
  import('@ansoragroup/ui').then(m => ({ default: m.ProductImageGallery as any }))
)

function ProductPage() {
  return (
    <Suspense fallback={<Loading />}>
      <ProductImageGallery images={images} />
    </Suspense>
  )
}
```

<SubTitle>3. i18n: only include needed locales</SubTitle>

The translation files are the largest part of the bundle (~20 KB gzip for all 3 locales). If your app only uses one locale, the other translation objects are tree-shaken away since they are not referenced by the component code directly -- they are loaded through the provider.

```tsx
// The GeekShopProvider only loads the locale you specify
<GeekShopProvider locale="uz" currency="UZS">
  <App />
</GeekShopProvider>
```

<SubTitle>4. Analyze your bundle</SubTitle>

Use your bundler's analysis tools to verify tree-shaking:

```bash
# Vite
npx vite-bundle-visualizer

# Webpack
npx webpack-bundle-analyzer stats.json

# Rollup
npx rollup-plugin-visualizer
```
